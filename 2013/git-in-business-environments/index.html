Git in der Praxis

Was Git für uns tun kann, sehen wir jetzt.

Git SVN

Initialisieren wir ein Subversion Repository mit Git:


git svn init -s http://svn.mycompany.com/svn/projectname

Dies erstellt ein leeres Git Repository, welches wir befüllen mit


git svn fetch

Nun ist das gesamte Repository lokal vorhanden.

Git SVN

Jeder Branch, jeder Tag ist lokal vorhanden:


git branch -a

Falls das geklonte Repository eine umfangreiche Historie hat, können wir es packen:


git repack -d

Im Anschluss daran ist die Historie komprimiert.

Git SVN

Arbeiten an einem neuen Feature:


git checkout -b neues_feature
git add path/to/new_file
git add path/to/edited_file
git add another/edited/file
git commit -m "commit message"

Fertig!

Git SVN

Das Feature in Subversion schieben:


git checkout master
git merge neues_feature
git svn rebase
git svn dcommit

Im Anschluss daran das Feature in Subversion.

Git: Konfiguration

Git möchte wissen, wer wir sind. Daher:


git config --global user.name "Your Name"
git config --global user.email "your_email@whatever.com"

Git: Konfuguration

Um zu vermeiden, dass Line Endings Probleme machen...


git config --global core.autocrlf input
git config --global core.safecrlf true

Windows User brauchen allerdings


git config --global core.autocrlf true
git config --global core.safecrlf true

Git: Ein neues Projekt

Um ein neues Projekt in ein Git Repository umzuwandeln:


cd MyProject
git init
Initialised empty Git repository in /home/...

Git: Änderungen machen

Änderungen am Projekt funktionieren wie bei Subversion:


git add -p
git rm
git mv
git reset --

Aber: Git beobachtet Änderungen einzeln, nicht ganze Dateien!

Git: Log und Diff

Mit Git stehen die vollständige Historie und Diffs jederzeit zur Verfügung.


git log
git show

Git: Alte Versionen anschauen

Es ist jederzeit möglich alte Versionen von Dateien, Ordern oder dem gesamten Projekt zu betrachten.


git checkout commit-id

Zurück in den aktuellen Stand gelangen wir mit:


git checkout master

Git: Tags

Tags erlauben uns Fixpunkte in der Historie zu setzen.


git tag
git tag -a v1.0.0
git checkout v1.0.0
git describe --tags
git push --tags
git tag -d v1.0.0

Git: Mitten im Feature ... und jetzt

Git bringt eine Zwischenablage mit.


git stash list
git stash save "message"
git stash apply stash@{0}
git stash pop
git stash drop commit-id
git stash clear
