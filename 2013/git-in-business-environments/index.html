Heads Up

First there was RCS, then there was CVS. Subversion succeeded CVS, and now git is there to become the new king in town. But will it blend in?

Subversion is everywhere, it is mainstream, and once was revolutionary just as git is considered to be the new thing, too. Even version control evolves, and grows into mainstream.

Subversion is missing out on

real branches

  1. no relation to the trunk
  2. merging loses the branch history
  3. branches are just copies

Subversion is missing out on

tagging release

  1. tags are no fix points
  2. they can be modified

Subversion is missing out on

proper file operations

  1. Bob moves a file
  2. Alice changes old file
  3. commit fails

merge conflicts are common

Subversion is fun

working offline is hard

  1. history, diffs, etc. are unavailable
  2. branching, tagging, commiting impossible

source trees are not pristine

developers are forced to live with it
